The Amazon Linux 2023 template is the actively maintained EKS-optimized AMI distribution. Pass `os_distro=al2023` to any `make` target to select this template.

<Warning>
Amazon EKS stopped publishing EKS-optimized Amazon Linux 2 (AL2) AMIs on November 26, 2025. Use AL2023 for all Kubernetes versions including 1.33 and higher.
</Warning>

## Template variables

The following variables are accepted by the AL2023 Packer template. Set defaults in a variable file or override them on the `make` command line. See [AMI template variables and build options](/building/overview) for how variable precedence works.

| Variable | Default | Description |
|----------|---------|-------------|
| `ami_name` | _(set by Makefile)_ | Name of the resulting AMI |
| `ami_description` | `EKS-optimized Kubernetes node based on Amazon Linux 2023` | AMI description |
| `ami_component_description` | `(k8s: <version>, containerd: <version>)` | Component version string appended to `ami_description` |
| `ami_regions` | _(empty)_ | Comma-separated list of additional regions to copy the AMI to |
| `ami_users` | _(empty)_ | AWS account IDs to share the AMI with |

| `binary_bucket_region` | `us-west-2` | Region of the binary S3 bucket |
| `containerd_version` | `2.*` | containerd version to install |
| `install_containerd_from_s3` | `false` | Install containerd from S3 instead of the OS repo |

## Instance type and volume size

The Makefile sets a default instance type based on the target architecture. You can override it for builds that need more resources, such as accelerated builds:

```bash
make build os_distro=al2023 k8s=1.31 instance_type=g4dn.xlarge
```

For accelerated AMIs, set `launch_block_device_mappings_volume_size` to at least `20` GiB (the default). If you install additional drivers or toolkits, consider increasing this:

```bash
make build os_distro=al2023 k8s=1.31 \
  enable_accelerator=nvidia \
  launch_block_device_mappings_volume_size=30
```

## SSH interface

By default, Packer connects to the builder instance over SSH using a temporary key pair and security group. In environments without public internet access, use AWS Systems Manager Session Manager instead:

```bash
make build os_distro=al2023 k8s=1.31 \
  ssh_interface=session_manager \
  subnet_id=subnet-0abc123
```

<Note>
When using `ssh_interface=session_manager`, the source AMI must not be the minimal variant, since the minimal AMI does not include the SSM agent.
</Note>

## Pause container image

The pause container image is baked into the AMI at build time. The `nodeadm` utility reads the `pause_container_image` variable to populate the `sandbox_image` field in the containerd `config.toml`. The image is not pulled at runtime.

The default image points to an EKS-owned ECR registry:

```
public.ecr.aws/eks-distro/kubernetes/pause:3.10
```

To use a different image, such as the upstream Kubernetes pause image:

```bash
make build os_distro=al2023 k8s=1.31 \
  pause_container_image=registry.k8s.io/pause:3.10
```

To use an ECR image in a different region, find the region-specific registry at the [EKS add-ons images reference](https://docs.aws.amazon.com/eks/latest/userguide/add-ons-images.html).

To use a FIPS-compliant ECR endpoint, change `ecr` to `ecr-fips` in the registry URI:

```diff
-<ACCOUNT_ID>.dkr.ecr.<REGION>.amazonaws.com
+<ACCOUNT_ID>.dkr.ecr-fips.<REGION>.amazonaws.com
```

## SSM agent version

The `ssm_agent_version` variable controls which version of the AWS Systems Manager agent is installed:

- **Empty (default)** — installs the latest version available in the Amazon Linux 2023 core repositories.
- **`latest`** — installs the latest version from the SSM agent S3 bucket managed by the SSM project.
- **Specific version** — installs an exact release from the SSM project S3 bucket.

```bash
# Install a specific SSM agent version
make build os_distro=al2023 k8s=1.31 ssm_agent_version=3.2.1630.0
```
