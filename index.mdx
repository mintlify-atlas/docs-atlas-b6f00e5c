Amazon EKS AMI is the same Packer-based build specification that AWS uses to create the official EKS-optimized AMIs. Use it to build custom nodes with your own configuration, Kubernetes version, or hardware acceleration requirements — and initialize them declaratively with `nodeadm`.

## Get started

<Steps>
  <Step title="Install prerequisites">
    Install [Packer](https://www.packer.io/) v1.8.0 or later and configure AWS credentials with the required IAM permissions.
  </Step>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/awslabs/amazon-eks-ami.git
    cd amazon-eks-ami
    ```
  </Step>
  <Step title="Build an AMI">
    ```bash
    make k8s=1.33 os_distro=al2023
    ```
    This builds an EKS-optimized AMI for Kubernetes 1.33 on Amazon Linux 2023 in `us-west-2`.
  </Step>
  <Step title="Configure your nodes">
    Use `nodeadm` and the `NodeConfig` API to initialize nodes at launch by providing configuration in your EC2 instance's user data.
  </Step>
</Steps>

<Note>
  Amazon EKS stopped publishing EKS-optimized Amazon Linux 2 (AL2) AMIs on November 26, 2025. AL2023 is the current supported distribution for all Kubernetes versions 1.33 and higher.
</Note>
